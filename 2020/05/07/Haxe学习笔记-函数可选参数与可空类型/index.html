Haxe学习笔记:函数可选参数与可空类型

2020-05-07 Views

一点小记

haxe中的函数声明允许修饰参数为可选参数，通过于 参数签名 前方加上?来表示该参数是可选的

但是有一点需要稍微注意，因为 haxe 的无平台特性，考虑到编译至静态目标平台（如：C++、Flash、Java、C#）不支持基本类型值为空 null 的情况，可选参数会被隐式包装为可空类型 Null<T> 。如上例子中的函数func类型为 Null<Int> -> String -> Void （为了满足函数式编程，Haxe 中的函数具有类型），在静态目标平台下可选参数会根据对应平台默认赋值，以下引用自官方手册：

定义：默认值

静态目标语言中，基本类型的默认值如下：
Int —— 0；
Float —— Flash 中为 NaN，其它静态目标语言中为 0.0 ；
Bool —— false 。

注意：实际上 Haxe 官方手册中定义的基本数据类型就只有这三种，String 不属于基本类型。

且没有被包装为 Null<T> 类型的基本类型，也不可与 null 值进行比较（静态语言平台下），同时如果有一个 null 值被隐藏于某个 Null<T> 类型或 Dynamic 类型中赋值给一个基本类型，则会使用默认值赋值给该基本类型变量。

定义：Null<T>

在静态目标语言中，可以使用 Null<Int>、Null<Float>、和 Null<Bool> 类型来允许 null 作为一个值。在动态目标语言，这样做没有影响。Null<T> 也可以被用于其它类型用以表示 null 是一个允许的值。
