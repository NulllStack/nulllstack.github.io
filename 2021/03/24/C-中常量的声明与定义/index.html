C++中常量的声明与定义

2021-03-24 Views

C++ Primer 中提到，对于需要在文件（编译单元）间共享的 const 变量，我们可以通过在其声明与定义前添加 extern 修饰符进行修饰以达到目的：

某些时候有这样一种 const 变量，它的初始值不是一个常量表达式，但又确实有必要在文件间共享。这种情况下，我们不希望编译器为每个文件分别生成独立的变量。相反，我们想让这类 const 对象像其他（非常量）对象一样工作，也就是说，只在一个文件中定义 const，而在其他多个文件中声明并使用它。

解决办法是，对于 const 变量不管是声明还是定义都添加 extern 关键字，这样只需定义一次就可以了：

// file_1.cc

extern const int bufSize = fcn();

//file_1.h

extern const int bufSize;

然而书中并没有进一步说明相比默认的 const 行为有哪些好处，而且看似 extern 修饰在声明与定义上都是必须的（然而不是）。 因此下面我结合自己的理解进一步讨论。

首先，const 变量默认都是 internal linkage （对内链接）的，这意味着它仅在自己所在的文件内有效，每个文件的 const 变量都是相互独立的。当一个头文件中定义了一个 const 变量，其他包含该头文件的各个文件都会各自为 const 以一个相同的值进行替换。这将带来一个问题，如果说该 const 变量是以某个函数的调用进行初始化：const int MY_CONST = getNum(); 那么由于每个包含了该头文件的文件都需要生成各自的 const ，就需要对 getNum() 这个函数进行展开，如此一来效率就会很低。如果该头文件中的 const 变量是以字面值进行初始化：const int MY_CONST = 5; 影响则不会很大。

因此我们在定义时可以对 const 变量进行 extern 修饰，这样会使得该 const 变量的定义转变为 external linkable （外部可链接）：extern const int MY_CONST = getNum(); 这会使得需要 external linkable 的声明可以找到该定义从而避免 getNum() 的替换。

而在其他需要使用到该 const 的文件我们通过 external linkage （外部链接）对 const 变量进行声明：extern const int MY_CONST; ，此时编译器便不会马上为 const 变量的初始化值进行替换，而是使其成为一个 external linkage 的声明，此时若 const 变量在自己的编译单元中找不到定义就会转而寻找 external linkable 的定义。

但是，声明可以有多份，定义只能有一份，当我们把 const 变量的定义放入头文件并以 extern 加以修饰之后，包含该头文件的文件便会出现重定义错误。为了解决这一问题，我们通常需要在头文件中以 extern 修饰 const 变量，不进行初始化，使其成为一个声明，并把实现放入另外的 .cpp 文件中，并且在该 .cpp 文件中包含声明所在的头文件，而其他同样包含该头文件的文件便不再需要对 const 进行声明：
